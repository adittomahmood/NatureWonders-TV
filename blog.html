JavaScript Concepts and REST API

Scope of Variables:

In JavaScript, variable scope determines the visibility and accessibility of variables. There are three types of variable scopes:

  • var: Variables declared with var have function-level scope. They are hoisted to the top of the function or global scope.
  • let: Variables declared with let have block-level scope, making them accessible only within the block they are defined in.
  • const: Variables declared with const also have block-level scope but cannot be reassigned after initialization.
It's recommended to use let and const over var in modern JavaScript.

Use Cases of null and undefined:

null and undefined are used to represent missing or absent values in JavaScript.

  • null: It explicitly represents the absence of any object value or a variable that intentionally has no value. It is often used to indicate that a variable should be empty or has no value.
  • undefined: It represents a variable that has been declared but has not been assigned a value. It can also be explicitly set to indicate a variable's unavailability.

REST API:

REST API (Representational State Transfer) is an architectural style for designing networked applications. It is based on a set of principles that enable stateless communication between client and server over HTTP.
Key principles of REST include:

  • Using standard HTTP methods (GET, POST, PUT, DELETE) for CRUD (Create, Read, Update, Delete) operations on resources.
  • Representing resources as nouns in the URL structure (e.g., /users for a user resource).
  • Using HTTP status codes to indicate the outcome of API requests (e.g., 200 for success, 404 for not found).
REST APIs are widely used for building web services, allowing different software systems to communicate over the internet in a simple and scalable manner.
